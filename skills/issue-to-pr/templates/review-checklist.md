# Reviewer Checklist Template

- [ ] The PR solves the issue described.
- [ ] The implementation is scoped and does not introduce unrelated changes.
- [ ] Public API or user-facing behavior changes are documented.
- [ ] Error, empty, loading, and edge states are handled where relevant.
- [ ] Tests cover the changed behavior.
- [ ] The change does not weaken security, privacy, or validation.
- [ ] The PR description clearly explains what changed and how it was tested.
