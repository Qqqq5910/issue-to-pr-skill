# Output Format Notes

Use standard GitHub Markdown for all generated plans and PR descriptions.

## Rules

- Use clear headings.
- Use normal bullet lists and numbered lists.
- Use task-list checkboxes for risk and reviewer items.
- Keep placeholders simple and copyable.
- Avoid nested fenced code blocks in the generated output unless the user explicitly asks for code samples.
- Keep the final PR description ready to paste into GitHub.

## Required sections

- Issue Summary
- Task Type
- Requirements and Constraints
- Likely Affected Areas
- Implementation Plan
- Risk Checklist
- Test Plan
- Pull Request Description
- Reviewer Checklist
