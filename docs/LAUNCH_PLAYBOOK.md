# Launch Playbook

This playbook helps turn `issue-to-pr-skill` into a discoverable open-source project.

## Positioning

One-liner:

> Turn messy GitHub issues into structured implementation plans, test plans, PR descriptions, and reviewer checklists for AI coding agents.

Core angle:

AI coding agents can edit code, but they still need workflow discipline. This skill acts as a planning and QA gate before code changes begin.

## Ideal users

- Open-source maintainers who receive vague issues
- Indie hackers using AI coding agents
- Claude Code, Cursor, and Codex users
- Contributors joining unfamiliar repositories
- Teams that want better PR descriptions and test plans

## Launch sequence

### Soft launch

Post a short before-after demo. Lead with the problem, not with the technology.

### Developer community launch

Share it as a reusable workflow tool for developers who already use AI coding agents. Adapt the post to each community and avoid copy-paste announcements.

### Follow-up content

Publish examples instead of repeated announcements:

- Messy issue to clean PR plan
- How to reduce AI coding agent over-engineering
- A tiny Markdown skill that improves PR review quality

## Success metrics

- GitHub stars
- Forks
- Issues opened by others
- README traffic
- Mentions in developer communities
- Requests for related skills

## What not to do

- Do not market it as a full coding agent.
- Do not promise automatic bug fixing.
- Do not add runtime dependencies before there is user demand.
- Do not spam the same post across communities.

## Expansion path

After the first feedback, expand into a developer workflow skill pack:

- `issue-to-pr`
- `bug-repro`
- `repo-onboarding`
- `test-writer`
- `pr-reviewer`
- `release-notes`
